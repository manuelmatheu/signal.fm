Changelog

Version history for signal.fm

v0.5
2026-04-16 Major
  • Last.fm picks fixed — the official Last.fm recommendation APIs were removed (error 3); rebuilt on Last.fm’s internal web player station endpoint, proxied through a Vercel serverless function (api/lfm-station.js) to bypass CORS
  • OAuth saves username — Last.fm OAuth now auto-stores your username from the session response; no manual entry needed to activate the picks signal
  • UI polish — removed a banned side-stripe CSS pattern from the now-playing indicator; redesigned the welcome screen (signal pills, left-aligned headline) and generate screen (cleaner typographic hierarchy); removed ~80 lines of dead CSS from the removed sidebar
  • Layout — feed column narrowed to 820px for comfortable reading width; established spatial rhythm between signal filters, feed context, and track list
v0.4
2026-04-02 Major
  • Ambient background — album art of the playing track bleeds into the full app background, blurred and saturated, crossfading between tracks (1.4s transition)
  • DM Serif Display — new display font for the logo, welcome headline, and feed context title; pairs with Manrope for body text
  • Cinematic player bar — glow shadow when playing, album art grows with accent glow, play button breathes with a subtle pulse animation
  • Now-playing indicator — accent left-border inset + album art glow on the active track row
  • Welcome CTA pulse — connect button animates with a slow glow loop when idle
  • Footer & changelog — persistent footer with version link opening this page as an overlay
v0.3
2026-04-01 Major
  • Neon Pulse design system — deep purple void dark theme, pastel lavender light theme, warm orange accent throughout
  • Manrope typography — replaced system fonts with Manrope 400–800
  • Signal toggles — per-signal checkboxes for artist similarity, track similarity, new releases, and Last.fm picks
  • Last.fm picks signal — personalized recommendations via Last.fm session auth (optional)
  • Feed context panel — collapsible liner notes showing seeds, active signals, and narrative context
  • Mobile hamburger drawer — sidebar becomes a slide-in drawer with backdrop on small screens
  • Like / skip feedback loop — heart boosts artist weight +0.3; skip softens it −0.2; feed adapts in real time
v0.2
2026-03-xx Feature
  • All four signals — artist similarity, track similarity, new releases, and daily rotation feeding a single deduplicated candidate pool
  • Infinite scroll — IntersectionObserver sentinel pre-generates the next batch while you listen
  • Candidate buffer — next batch pre-built in the background to keep scroll seamless
  • Artist ID cache — Last.fm artist names resolved to Spotify IDs and cached for 24h
  • Save as Playlist — export the current session feed as a private Spotify playlist
v0.1
2026-03-xx Launch

Initial build. Spotify PKCE OAuth, Web Playback SDK, seed pool from top artists, artist similarity signal via Last.fm, basic feed rendering and player bar.
